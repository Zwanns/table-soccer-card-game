# Задание для Codex: добавить национальные рубашки колод

## Контекст

Проект:

```text
Total Soccer: Mundial
```

Стек:

```text
TypeScript
Phaser 3
Vite
Vitest
```

Перед матчем игроки выбирают национальные сборные.

Флаги стран уже находятся в папке:

```text
public/flags/
```

Необходимо добавить отдельные изображения рубашек колод для каждой сборной.

Рубашки будут находиться в новой папке:

```text
public/covers/
```

---

# Цель задачи

На игровом экране матча вместо текущей стандартной рубашки колоды показывать индивидуальную рубашку выбранной сборной.

Для каждого игрока нужно использовать рубашку той страны, которую он выбрал перед матчем.

Пример:

```text
Игрок 1 выбрал Poland
Игрок 2 выбрал Argentina
```

На экране матча:

```text
колода игрока 1 -> public/covers/poland.webp
колода игрока 2 -> public/covers/argentina.webp
```

Если подходящего файла рубашки нет, использовать fallback:

```text
public/covers/none.webp
```

---

# Правила именования файлов

Названия файлов рубашек соответствуют названиям файлов флагов в:

```text
public/flags/
```

Но формат изображения рубашки всегда:

```text
.webp
```

Нужно брать имя файла флага без расширения и добавлять расширение:

```text
.webp
```


Не хардкодить названия стран вручную.

Использовать существующие данные сборных из:

```text
src/data/nationalTeams.ts
```

# Размер исходных изображений

Все изображения рубашек создаются в разрешении:

```text
848 × 1344 px
```

Соотношение сторон:

```text
848 / 1344
```

Приложение должно масштабировать изображение пропорционально.

Запрещено:

```text
растягивать изображение по ширине или высоте независимо
искажать пропорции
обрезать важные части изображения без необходимости
```

Для визуального отображения рубашки использовать пропорциональное масштабирование с сохранением исходного aspect ratio.

Предпочтительный режим:

```text
contain
```

То есть изображение полностью помещается в доступную область карты, центрируется и не искажается.

# Обработка отсутствующего файла

Если файла рубашки выбранной сборной нет, игра не должна ломаться.

Пример:

```text
public/covers/nigeria.webp отсутствует
```

Ожидаемое поведение:

```text
использовать public/covers/none.webp
не показывать ошибку пользователю
не останавливать загрузку сцены
не ломать переход в матч
```

Допускается записать техническое предупреждение в console:

```ts
console.warn(
  `Cover not found for ${team.name}. Falling back to covers/none.webp.`,
);
```

Нужно корректно обработать ошибку загрузки Phaser Loader.

Не делать запрос файла рубашки заново при каждом рендере динамического слоя.

# Пропорциональное масштабирование

Добавить вспомогательную функцию для масштабирования изображения по принципу `contain`.

Пример:

```ts
export function fitImageContain(
  image: Phaser.GameObjects.Image,
  bounds: {
    width: number;
    height: number;
  },
): void {
  const scale = Math.min(
    bounds.width / image.width,
    bounds.height / image.height,
  );

  image.setScale(scale);
}
```

Изображение должно:

```text
сохранять исходные пропорции
быть центрировано
полностью помещаться в область визуальной карты колоды
не выходить за пределы карты
```

Если компонент колоды использует обводку, тени или эффект стопки карт, сохранить их.

# UX-требования

1. Рубашка должна хорошо читаться даже при небольшом размере колоды.
2. Изображение нельзя искажать.
3. При отсутствии национальной рубашки игрок не должен видеть ошибку.
4. Fallback `none.webp` должен выглядеть как обычная нейтральная рубашка.
5. Загрузка отсутствующего файла не должна ломать матч.
6. Не добавлять новые надписи поверх игрового поля.
7. Не менять размер или расположение колод без необходимости.

---

# Критерии приемки

Задача считается выполненной, если:

1. Создана папка:

```text
public/covers/
```

2. Приложение использует национальные рубашки из:

```text
public/covers/
```

3. Название файла рубашки автоматически определяется по basename файла флага сборной.
4. Формат рубашек:

```text
.webp
```

5. Для отсутствующей рубашки используется:

```text
public/covers/none.webp
```

6. Ошибка загрузки национальной рубашки не ломает матч.
7. Для двух выбранных сборных могут одновременно отображаться две разные рубашки.
8. Изображения масштабируются пропорционально.
9. Существующий UI колод сохраняется.
10. Игровые правила не изменены.
11. Все тесты проходят.
12. Проект успешно собирается.

# Проверка после реализации

Выполнить:

```bash
npm test
npm run build
```

Также вручную проверить через:

```bash
npm run dev
```

Сценарии ручной проверки:

```text
1. Выбрать две страны, для которых существуют covers/*.webp.
   Убедиться, что отображаются две разные национальные рубашки.

2. Выбрать страну, для которой covers/*.webp отсутствует.
   Убедиться, что отображается covers/none.webp.

3. Начать матч.
   Убедиться, что колода остается кликабельной.

4. Взять атакующую карту.
   Убедиться, что игровая логика не изменилась.

5. Вернуться в меню и начать новый матч с другими странами.
   Убедиться, что рубашки обновились.
```
