# Total Soccer: Mundial
# Добавить режим игры против AI

## Цель

Добавить честного эвристического компьютерного соперника без уровней сложности.

AI должен:
- использовать только легальные действия;
- видеть только ту же открытую информацию, что доступна человеку;
- не подсматривать карты в закрытых колодах;
- не знать будущие карты;
- не использовать seed для просмотра будущего;
- принимать понятные решения;
- иметь небольшую вариативность поведения;
- работать в быстром матче;
- работать в турнирном режиме;
- поддерживать матчи:
  - HUMAN vs HUMAN;
  - HUMAN vs AI;
  - AI vs HUMAN;
  - AI vs AI.

Выполнять работу строго по этапам.

После каждого этапа запускать:

```bash
npm test
npm run build
````

Если менялся UI или runtime:

```bash
npm run dev
```

После каждого этапа остановиться и вывести отчет.

# 5. Эвристика AI в фазе WAITING_FOR_ATTACK_CARD

## Условие

Если:

```text
state.phase === 'WAITING_FOR_ATTACK_CARD'
```

AI выбирает источник атаки.

## Если текущая линия не MIDFIELD

Всегда:

```ts
{
  type: 'DRAW_FROM_DECK'
}
```

## Если текущая линия MIDFIELD

AI может выбрать:

```text
DRAW_FROM_DECK
```

или:

```text
COMMIT_MIDFIELDER
```

## Правила выбора полузащитника

Использовать только:

```text
state.committableMidfielderPositionIds
```

Для каждого допустимого полузащитника:

1. взять открытую карту собственного midfield-slot;
2. взять открытую карту соперника строго напротив;
3. проверить возможность пробития через существующие `cardRules`;
4. не использовать диагональные коридоры;
5. не подключать полузащитника против пустого слота;
6. не использовать слот повторно в одной атаке.

## Базовая стратегия

Если есть успешно пробивающий полузащитник:

* в большинстве случаев подключить его;
* иногда выбрать deck для вариативности;
* среди нескольких успешно пробивающих полузащитников предпочитать минимально достаточную карту;
* среди равноценных вариантов выбирать через seeded random.

Рекомендуемая вероятность подключения:

```text
70%
```

Рекомендуемая корректировка:

```text
если AI проигрывает по счету -> 80%;
если AI ведет в счете -> 55%;
если счет равный -> 70%.
```

Если успешно пробивающего полузащитника нет:

```ts
{
  type: 'DRAW_FROM_DECK'
}
```

## Важно

AI не может смотреть верхнюю карту deck до выбора:

```text
DRAW_FROM_DECK
```

# 6. Эвристика AI в фазе WAITING_FOR_TARGET

## Условие

Если:

```text
state.phase === 'WAITING_FOR_TARGET'
```

AI должен выбрать цель.

## Приоритет 1 — открытая зона контратаки

Если:

```text
state.legalMidfieldGapPositionIds.length > 0
```

и текущая карта пришла из:

```text
DECK
```

AI должен использовать gap.

Выбрать один допустимый slot:

* seeded random среди доступных;
* не использовать gap для подключенного полузащитника;
* не использовать gap для GK.

Вернуть:

```ts
{
  type: 'SELECT_MIDFIELD_GAP',
  positionId,
}
```

## Приоритет 2 — пробиваемая обычная цель

Если gap недоступен:

1. взять:

   ```text
   state.legalTargetPositionIds
   ```
2. определить пробиваемые цели через существующие `cardRules`;
3. если есть несколько пробиваемых целей:

   * предпочитать цель с наиболее сильной захватываемой картой;
   * среди равноценных использовать seeded random.

Вернуть:

```ts
{
  type: 'SELECT_TARGET',
  positionId,
}
```

## Приоритет 3 — обязательная неудачная попытка

Если пробиваемых целей нет:

* выбрать одну легальную цель;
* предпочитать цель с минимальным rank;
* среди равноценных использовать seeded random.

Вернуть:

```ts
{
  type: 'SELECT_TARGET',
  positionId,
}
```

Это имитирует существующую механику без кнопки `OUT`:

* AI нажимает на непробиваемую карту;
* создается стандартный turnover;
* отдельная логика завершения атаки не добавляется.

## GK

При линии GK:

* использовать единственную легальную GK-цель;
* не добавлять отдельную AI-логику сэйва или штанги;
* результат определяет `GameEngine`.

# 7. Seeded random для решений AI

## Требование

Не использовать:

```ts
Math.random()
```

## Создать отдельный AI random stream

AI-решения не должны менять порядок карт в deck или GK-deck.

Использовать отдельный seeded random stream, производный от:

```text
match seed;
playerId;
фиксированного namespace AI.
```

Пример идеи:

```ts
createSeededRandom(
  `${matchSeed}:AI:${playerId}`,
)
```

Адаптировать под существующий формат seed.

## Требования

```text
одинаковый seed -> одинаковые AI-решения;
действия AI не меняют shuffle основной deck;
действия AI не меняют shuffle GK-deck;
разные seed могут давать разные решения среди равноценных вариантов.
```
