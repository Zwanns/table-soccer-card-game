# 3. Цель задачи

Добавить возможность подключать карты собственной линии полузащиты к атаке.

Во время прохождения линии полузащиты соперника активный игрок вместо взятия очередной атакующей карты из основной колоды может нажать на одного из своих полузащитников.

Подключенный полузащитник:

1. Снимается со своей позиции на поле.
2. Становится текущей атакующей картой.
3. Автоматически атакует только карту соперника, расположенную строго напротив.
4. При успешном пробитии участвует в существующем жизненном цикле обычной атакующей карты.
5. При неудаче приводит к потере мяча.
6. Оставляет после себя временно пустой слот.

За одну атаку игрок может подключить до трех своих полузащитников: по одному из каждого слота.

После неудачной атаки все освобожденные слоты остаются пустыми во время немедленной контратаки соперника.

Независимо от количества пустых слотов соперник получает только одну общую возможность провести одну обычную карту через любую из этих открытых зон.

# 4. Термины

Использовать стабильные идентификаторы позиций:

midfielder-1
midfielder-2
midfielder-3

Не использовать в логике понятия:

left
right
левый
правый

Причина: команды находятся на противоположных сторонах поля, поэтому визуальные направления могут трактоваться неоднозначно.

## 4.1 Подключенный полузащитник

Обычная полевая карта собственной линии midfield, которую активный игрок снимает со своего поля и использует как атакующую карту.

## 4.2 Освобожденный слот

Пустая позиция midfield, которая появилась после подключения собственного полузащитника к атаке.

## 4.3 Открытая зона

Временная возможность соперника провести одну обычную атакующую карту через любой освобожденный слот midfield во время немедленной контратаки.

Важно:

Открытая зона не является отдельной картой.
Открытая зона не является виртуальной картой rank 2.
Открытая зона не участвует в cardRules.

# 5. Соответствие коридоров полузащиты

Каждый подключенный полузащитник может атаковать только карту соперника с тем же positionId:

midfielder-1 -> midfielder-1
midfielder-2 -> midfielder-2
midfielder-3 -> midfielder-3

Запрещены любые диагональные атаки:

midfielder-1 -> midfielder-2
midfielder-1 -> midfielder-3

midfielder-2 -> midfielder-1
midfielder-2 -> midfielder-3

midfielder-3 -> midfielder-1
midfielder-3 -> midfielder-2

Если в противоположном слоте соперника нет фактически лежащей карты, использовать своего полузащитника из этого коридора нельзя.

Единое правило:

Нет карты напротив -> нет возможности подключить полузащитника этого коридора.

Не имеет значения, почему слот соперника пуст:

1. Карта ранее была побита.
2. Полузащитник соперника был подключен к предыдущей атаке.
3. Слот не удалось восстановить из-за пустой основной колоды.

Подключенный полузащитник может атаковать только существующую карту соперника.

# 6. Когда можно подключить полузащитника

Активный игрок может выбрать своего полузащитника вместо клика по основной колоде, если одновременно выполняются условия:

1. Текущая фаза соответствует ожиданию следующей атакующей карты:
   WAITING_FOR_ATTACK_CARD
   или фактический аналог в актуальном коде.

2. Атака еще находится на уровне линии midfield соперника.

3. Очередная атакующая карта еще не взята из основной deck.

4. В собственном выбранном слоте лежит обычная полевая карта.

5. В противоположном слоте midfield соперника лежит обычная полевая карта.

6. Выбранная карта еще не использовалась в текущей атаке.

7. Текущая линия соперника еще не пройдена.

8. В данный момент не выполняется анимация.

Не ограничивать кликабельность только выигрышными вариантами.

Если собственный полузащитник не может побить карту напротив, игрок все равно имеет право нажать на него и сознательно проиграть дуэль.

Не добавлять визуальную подсказку, заранее раскрывающую результат сравнения карт.

После выхода атаки за линию midfield обычные правила продолжают работать без изменений.

# 9. Использование полузащитника: успешная дуэль

При клике по собственному полузащитнику:

1. Определить противоположную цель по тому же positionId.
2. Создать animation context до мутации состояния.
3. Снять выбранную карту со своего поля.
4. Оставить собственный слот пустым.
5. Зафиксировать слот как освобожденный подключением.
6. Использовать выбранную карту как обычную текущую атакующую карту.
7. Проверить пробитие через существующий cardRules.
8. Не копировать и не переписывать правила сравнения карт.

Если атакующая карта может побить карту соперника:

1. Выполнить стандартную обработку успешного пробития обычной полевой карты.
2. Удалить карту соперника с поля.
3. Создать существующее событие:
   CARD_DEFEATED
4. Направить карты по существующему жизненному циклу attackBank.
5. Сохранить существующую механику перекрашивания захваченных карт.
6. Продолжить атаку.
7. Вернуться к ожиданию выбора следующего источника атакующей карты:
   - основная deck;
   - другой доступный собственный полузащитник.

Не создавать специальную альтернативную систему утилизации обычных карт.

Подключенный полузащитник после использования не возвращается автоматически в свой слот.

# 10. Использование полузащитника: неудачная дуэль

Если подключенный полузащитник не может побить карту соперника напротив:

1. Собственный полузащитник уже снят с поля.
2. Его слот остается пустым.
3. Карта соперника остается на поле.
4. Подключенный полузащитник направляется по существующей логике завершения неудачной атаки.
5. Создается:
   ATTACK_MISSED
   или фактический существующий аналог.
6. Атака завершается.
7. Карты attackBank возвращаются вниз основной колоды активного игрока согласно существующим правилам.
8. Ход переходит сопернику.
9. Показывается существующее сообщение:
   Мяч потерян...
10. Не воспроизводится звук GK.
11. Игрок не может выбрать другую цель или другую карту после проигранной дуэли.

Первый клик является окончательным решением.

# 11. Когда возникает открытая зона

Открытая зона создается только при выполнении всех условий:

1. Игрок подключил минимум одного собственного полузащитника.
2. После этого его атака завершилась без гола.
3. Немедленно начинается ответная атака соперника.
4. Освобожденные позиции midfield остаются пустыми.

Причинами завершения атаки без гола могут быть:

1. Неудачная дуэль обычной картой.
2. Неудачная дуэль подключенным полузащитником.
3. Сейв GK.
4. Штанга.
5. Иной существующий не-голевой сценарий завершения атаки.

Важно:

Открытая зона создается только из слотов, освобожденных подключением полузащитников.

Не считать открытыми зонами любые произвольные пустые позиции midfield, которые появились по другим причинам.

# 12. Количество открытых зон

Физически пустыми могут остаться:

1. Один слот.
2. Два слота.
3. Три слота.

Но активный игрок ответной атаки получает только одну общую возможность провести карту через открытую зону.

Пример:

Полузащита соперника:

midfielder-1 = empty
midfielder-2 = empty
midfielder-3 = empty

Эти три пустых слота не дают три отдельных прохода.

Они дают один общий ресурс:

one counterattack midfield gap

Игрок может выбрать любой из трех пустых слотов только один раз.

После первого использования:

1. Все оставшиеся временно пустые слоты продолжают визуально оставаться пустыми.
2. Но ни один из них больше не является кликабельным как открытая зона в рамках этой атаки.
3. Повторно сбросить в них другие карты нельзя.

Запрещено:

3. Создавать CARD_DEFEATED при проходе открытой зоны.
4. Добавлять несуществующую defenderCard в attackBank.
5. Учитывать открытую зону как побитую карту.
6. Искусственно увеличивать статистику пробитых карт.
7. Искусственно увеличивать advantage или possession только из-за прохода gap.

Открытая зона — отдельный игровой случай.

# 14. Когда можно использовать открытую зону

Открытую зону может использовать только обычная атакующая карта, взятая из основной deck.

Нельзя использовать открытую зону:

1. Подключенным полузащитником.
2. GK-картой.
3. Без текущей атакующей карты.
4. После завершения текущей атаки.
5. Повторно в рамках той же контратаки.
6. После закрытия окна использования открытой зоны.

Открытая зона является дополнительной необязательной возможностью.

Игрок не обязан использовать ее.

# 15. Окно использования открытой зоны

Добавить явное временное окно для единой открытой зоны.

Рекомендуемое состояние:

type CounterattackMidfieldGap = {
  defenderId: PlayerId;
  positionIds: MidfielderPositionId[];
  used: boolean;
  windowOpen: boolean;
};

Названия можно адаптировать к текущему стилю проекта.

Правила окна:

1. Окно открывается в начале немедленной ответной атаки, если у защищающейся команды есть минимум один слот, освобожденный подключением полузащитника.

2. Пока в линии midfield соперника остаются обычные карты:
   - игрок может выбрать обычную карту midfield;
   - либо один раз использовать открытую зону.

3. Если обычных карт midfield больше нет:
   - открытая зона может быть использована как необязательный дополнительный проход перед первой атакой по defense;
   - либо игрок может сразу атаковать defense и отказаться от использования gap.

4. Если игрок впервые атакует карту линии defense, неиспользованное окно gap закрывается.

5. Если атака доходит до goalkeeper, окно gap уже должно быть закрыто.

6. Если атака заканчивается по любой другой причине, окно закрывается.

Это правило необходимо, чтобы сценарий с тремя снятыми полузащитниками работал корректно:

- линия midfield физически пуста;
- соперник все равно получает ровно одну возможность отправить одну слабую карту в любой свободный слот;
- но может отказаться от этого и сразу атаковать defense.

# 16. Проход через открытую зону

Если обычная атакующая карта из основной deck проходит открытую зону:

1. Игрок нажимает на любой доступный пустой слот midfield соперника.
2. Создать animation context до мутации состояния.
3. Атакующая карта летит:
   from active main deck
   ->
   to selected empty midfield slot
4. Показать короткий мягкий визуальный импульс.
5. Не показывать эффект побитой карты.
6. Не воспроизводить звук GK.
7. Не создавать CARD_DEFEATED.
8. Не создавать defenderCard.
9. Направить текущую обычную атакующую карту по существующему жизненному циклу attackBank.
10. Зафиксировать:
    used = true
    windowOpen = false
11. Продолжить атаку.
12. Вернуться к ожиданию следующей обычной атакующей карты.

После использования gap:

1. Пустой слот остается визуально пустым.
2. Карта не остается лежать в этом слоте.
3. Остальные пустые слоты не становятся доступными для повторного прохода.

# 17. Восстановление пустых слотов после неудачной атаки

Если игрок подключил полузащитников и не забил гол:

1. Его освобожденные слоты остаются пустыми во время немедленной ответной атаки соперника.
2. Соперник получает одно окно открытой зоны.
3. После завершения ответной атаки окно открытой зоны полностью удаляется.
4. При начале следующего хода владельца поля пустые позиции восстанавливаются стандартной логикой fieldRules.
5. Для восстановления используются только обычные карты основной deck.
6. GK-колода не участвует.
7. Если обычных карт в deck недостаточно, восстановить столько позиций, сколько возможно.
8. Остальные слоты оставить пустыми без ошибки.

Проверить актуальный flow startNextTurn и restoreField.

Если существующая логика уже естественным образом восстанавливает поле только активного игрока в начале его хода, не добавлять лишний механизм отложенного восстановления.

Добавлять отдельное deferred-состояние только если оно действительно требуется фактической реализацией.

# 18. Исключение: восстановление после гола

Если игрок подключил одного или нескольких полузащитников и в рамках этой же атаки забил гол:

1. Выполнить существующую голевую логику.
2. Не смешивать обычные карты с GK-картами.
3. Сохранить существующий рециклинг GK защищавшейся команды:
   GOALKEEPER_CARD_RECYCLED
4. После стандартного завершения attackBank восстановить освобожденные слоты midfield забившего игрока картами из его основной deck.
5. Восстанавливать только слоты, освобожденные подключением полузащитников в этой атаке.
6. Использовать стабильный порядок:
   midfielder-1
   midfielder-2
   midfielder-3
7. Для каждого восстановленного слота использовать существующее событие:
   FIELD_CARD_RESTORED
8. По возможности переиспользовать существующую анимацию:
   from main deck
   ->
   to restored field position
9. Если карт недостаточно, заполнить столько слотов, сколько возможно.
10. Оставшиеся позиции сохранить пустыми без ошибки.

# 27. Статистика, advantage и possession

Сохранить существующую модель.

## 27.1 Успешная дуэль подключенного полузащитника

Так как создается стандартное CARD_DEFEATED для positionId = midfielder-N:

1. Учитывается как обычное успешное пробитие midfield.
2. Может влиять на максимальную глубину атаки:
   midfield = 1
3. Не требует отдельной формулы.

## 27.2 Проход открытой зоны

MIDFIELD_GAP_USED:

1. Не является CARD_DEFEATED.
2. Не считается пробитием карты.
3. Не увеличивает число побитых карт.
4. Не создает фиктивную глубину.
5. Не должен сам по себе изменять possession.
6. Не должен сам по себе изменять advantage.

Если после gap игрок доходит до defense или goalkeeper, более глубокое продвижение учитывается существующими событиями.

# 31. Критерии приемки

Задача выполнена, если:

1. В фазе ожидания атакующей карты игрок может выбрать deck или допустимого собственного полузащитника.
2. Каждый полузащитник атакует только карту строго напротив.
3. Против пустого слота подключить полузащитника нельзя.
4. Можно подключить до трех разных полузащитников за одну атаку.
5. Можно чередовать полузащитников и карты из deck.
6. После прохода midfield подключение закрывается.
7. Успешная дуэль использует стандартный CARD_DEFEATED.
8. Неудачная дуэль завершает атаку как потерю мяча.
9. Турнир и пенальти не сломаны.
10. После не-голевой атаки освобожденные слоты остаются пустыми во время немедленной контратаки.
11. Независимо от числа пустых слотов создается только один общий gap.
12. Gap можно использовать максимум один раз.
13. Gap необязателен.
14. Gap можно пройти только обычной картой из основной deck.
15. Gap не является виртуальной картой 2.
16. Gap не создает CARD_DEFEATED.
17. Gap сам по себе не меняет advantage и possession.
18. После гола committed slots восстанавливаются обычными картами из основной deck.
19. GK-колода не смешивается с основной deck.
20. GK-карта не попадает в attackBank.
21. Гол, сейв, штанга и GK recycle продолжают работать.
22. Экипировки, составы, tooltip и шрифты не изменены.
23. npm run validate:kits проходит.
24. npm test проходит.
25. npm run build проходит.
