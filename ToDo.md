# Задание для Codex: подготовить архитектуру стилизованного главного меню Total Soccer: Mundial

## Контекст проекта

Проект:

```text
Total Soccer: Mundial
```

Стек:

```text
TypeScript
Phaser 3
Vite
Vitest
```

Главная сцена меню:

```text
src/scenes/MenuScene.ts
```

Игровая сцена:

```text
src/scenes/GameScene.ts
```

Размер сцены:

```text
SCENE_WIDTH = 1600
SCENE_HEIGHT = 720
```

Главное меню должно оставаться именно игровым меню, а не маркетинговым лендингом.

Нужно подготовить удобную архитектуру стилизованного главного меню, чтобы позже можно было легко заменить фон, логотип, декоративные элементы и добавить легкие анимации без переписывания всей сцены.

---

# Цель задачи

Переработать `MenuScene.ts` так, чтобы меню состояло из независимых визуальных слоев:

```text
1. background
2. overlay
3. decor
4. title / logo
5. buttons
6. footer / version
```

Основной принцип:

```text
Не делать все меню одной цельной картинкой.
```

Фон стадиона или футбольного поля должен быть отдельным ассетом.

Поверх него должны отдельно отрисовываться:

```text
логотип или текстовое название игры
кнопки
версия игры
мяч
флаги или декоративные элементы
легкие визуальные эффекты
```

---

# Визуальная концепция

Подготовить меню в стиле:

```text
Ночной футбольный стадион
```

или:

```text
Стилизованное футбольное поле с атмосферой большого турнира
```

Фон должен поддерживать тему:

```text
Mundial
```

Но не должен быть перегруженным.

Желательное настроение:

```text
футбол
турнир
стадион
вечерний свет
международная атмосфера
```

Не добавлять лишние элементы, которые мешают читать кнопки.

---

# Общая структура сцены

Организовать `MenuScene.ts` примерно так:

```ts
create(): void {
  this.createBackground();
  this.createOverlay();
  this.createDecor();
  this.createTitle();
  this.createButtons();
  this.createFooter();
  this.playIntroAnimation();
}
```

Допускается адаптировать названия методов под текущий стиль проекта.

Важно:

```text
не создавать одну длинную функцию create()
не смешивать загрузку ассетов, layout и анимации в одном блоке
не дублировать координаты без необходимости
```

---

# Этап 1. Фоновый слой

Добавить поддержку отдельного фонового ассета:

```text
public/menu/menu-bg.webp
```

или:

```text
public/menu/menu-bg.png
```

Предпочтительно:

```text
WEBP
```

если он нормально поддерживается текущей сборкой и браузерами проекта.

Фон должен:

```text
занимать весь экран 1600 x 720
масштабироваться через cover-like поведение
оставаться центрированным
не содержать встроенных кнопок
не содержать встроенного текста интерфейса
```

Добавить загрузку фонового ассета в:

```text
src/scenes/BootScene.ts
```

Например:

```ts
this.load.image('menu-bg', 'menu/menu-bg.webp');
```

Если ассет временно отсутствует, реализовать fallback:

```text
однотонный темно-зеленый или темно-синий фон
```

чтобы сцена не ломалась.

# Этап 2. Затемняющий overlay

Добавить отдельный затемняющий слой поверх фонового изображения.

Он нужен, чтобы:

```text
название игры хорошо читалось
кнопки не терялись на фоне
фон не был слишком ярким
```

Использовать Phaser Rectangle или Graphics, а не отдельную картинку.

Примерно:

```ts
this.add
  .rectangle(
    SCENE_WIDTH / 2,
    SCENE_HEIGHT / 2,
    SCENE_WIDTH,
    SCENE_HEIGHT,
    0x000000,
    0.35,
  );
```

Допускается подобрать другое значение alpha после визуальной проверки.

Не делать фон слишком темным.

---

# Этап 3. Декоративные элементы

Подготовить возможность добавлять отдельные декоративные спрайты:

```text
public/menu/ball.png
public/menu/flags-strip.png
public/menu/logo.png
```

На первом этапе:

```text
ball.png
flags-strip.png
logo.png
```

могут быть отсутствующими или временными.

Сцена должна корректно работать и без них.

Добавить централизованную настройку декоративных ассетов:

```ts
const MENU_ASSETS = {
  background: 'menu-bg',
  logo: 'menu-logo',
  ball: 'menu-ball',
  flags: 'menu-flags',
} as const;
```

Также создать отдельный метод:

```ts
createDecor(): void
```

В нем позже можно будет добавлять:

```text
мяч
полосу флагов
световые блики
декоративные карты
мягкие частицы
```

На первом этапе не перегружать сцену.

# Этап 4. Логотип или название игры

Сохранить поддержку текущего текстового названия:

```text
Total Soccer: Mundial
```

Но подготовить архитектуру так, чтобы позже можно было заменить текст на PNG-логотип.

Логика:

```text
если загружен menu-logo -> показывать logo sprite
иначе -> показывать Phaser Text
```

Текстовый fallback должен оставаться рабочим.

Расположение:

```text
центр по X
верхняя часть экрана
примерно y = 145
```

Можно использовать две строки:

```text
Total Soccer
Mundial
```

или одну строку, если так лучше соответствует текущему дизайну.

Название не должно перекрывать кнопки.

---

# Этап 5. Кнопки меню

Кнопки должны быть отдельными интерактивными UI-элементами.

Не встраивать их в фон.

Сохранить текущую логику переходов сцены.

Основная кнопка:

```text
Играть
```

или текущий существующий аналог.

При нажатии:

```ts
this.scene.start('TeamSelectScene');
```

Если в проекте уже существуют другие кнопки, сохранить их текущие действия.

Кнопки расположить вертикально по центру.

Примерный layout:

```text
x = 800
startY = 330
gap = 62
```

Каждая кнопка должна:

```text
быть читаемой
иметь hover-состояние
иметь disabled-состояние при необходимости
не сливаться с фоном
```

Можно использовать текущий компонент:

```text
src/ui/Button.ts
```

Если он уже подходит.

Не переписывать компонент без необходимости.

# Этап 6. Footer и версия

Добавить отображение версии игры в нижней части экрана:

```text
v0.6a
```

Источник значения:

```text
GAME_VERSION
```

из:

```text
src/config.ts
```

Не хардкодить строку версии напрямую в `MenuScene.ts`.

Расположение:

```text
x = 24
y = SCENE_HEIGHT - 24
```

или:

```text
x = SCENE_WIDTH - 24
y = SCENE_HEIGHT - 24
```

Выбрать вариант, который лучше выглядит визуально.

Текст должен быть небольшим и ненавязчивым.

# Этап 7. Легкие анимации

Добавить только ненавязчивые idle-анимации.

Разрешено:

```text
плавное появление логотипа
плавное появление кнопок
мягкое покачивание мяча
легкое изменение alpha у декоративных элементов
небольшой hover glow у кнопок
```

Не добавлять:

```text
яркие вспышки
агрессивные частицы
быстрые движения камеры
сильный parallax
постоянное мельтешение
резкие масштабирования
```

Пример для мяча:

```ts
this.tweens.add({
  targets: ball,
  y: ball.y - 8,
  duration: 1800,
  yoyo: true,
  repeat: -1,
  ease: 'Sine.easeInOut',
});
```

Пример для intro:

```ts
this.tweens.add({
  targets: title,
  alpha: 1,
  duration: 500,
  ease: 'Sine.easeOut',
});
```

# Этап 10. Не ломать текущий игровой flow

Проверить, что после изменений:

```text

кнопка Играть ведет на TeamSelectScene
выбор команд продолжает работать
GameScene открывается корректно
возврат из GameScene в MenuScene работает
версия отображается
```

Не менять:

```text
GameScene
GameEngine
правила карт
правила матча
экран результата
```

если это не требуется для меню.

# Архитектурные ограничения

Не делать:

```text
один большой JPG/PNG с уже нарисованными кнопками
текст кнопок внутри фонового изображения
hardcoded version string
дублирование SCENE_WIDTH и SCENE_HEIGHT
зависимость MenuScene от GameEngine
тяжелые анимации
сложную систему частиц на первом этапе
```

Сохранить возможность позже:

```text
заменить фон
заменить логотип
добавить флаги
добавить мяч
добавить декоративные карты
добавить более сложные анимации
```

без переписывания всей сцены.

# Рекомендуемая реализация fallback

Если декоративные ассеты пока не добавлены, меню должно показывать:

```text
темный фон
overlay
текстовое название
рабочие кнопки
версию
```

Для проверки наличия текстуры можно использовать:

```ts
this.textures.exists('menu-logo')
```

Пример:

```ts
if (this.textures.exists('menu-logo')) {
  this.add.image(centerX, titleY, 'menu-logo');
} else {
  this.add.text(centerX, titleY, GAME_TITLE, textStyle).setOrigin(0.5);
}
```

# Критерии приемки

Задача считается выполненной, если:

1. `MenuScene.ts` разделен на логические методы.
2. Есть отдельный фоновый слой.
3. Есть отдельный затемняющий overlay.
4. Кнопки остаются отдельными UI-элементами.
5. Название игры остается отдельным объектом.
6. Версия берется из `GAME_VERSION`.
7. Подготовлена папка `public/menu/`.
8. Добавлен `public/menu/README.md`.
9. Меню работает даже без дополнительных декоративных ассетов.
10. Нет одной цельной картинки с встроенными кнопками.
11. Не изменена логика матчей.
12. `npm test` проходит.
13. `npm run build` проходит.
