# Задание для Codex: убрать кнопку OUT и завершать неудачную атаку кликом по карте соперника

## Контекст проекта

Проект: `Total Soccer: Mundial`

Стек:

```text
TypeScript
Phaser 3
Vite
Vitest
```

Основная игровая сцена:

```text
src/scenes/GameScene.ts
```

Основной движок матча:

```text
src/game/GameEngine.ts
```

На текущем игровом экране есть кнопка:

```text
OUT
```

Сейчас она используется активным игроком для завершения атаки, если текущая атакующая карта не может побить карты актуальной линии соперника.

При выходе на вратаря кнопка `OUT` отключается, а игрок обязан нажать на карту голкипера, чтобы получить один из результатов:

```text
GOAL_SCORED
GOALPOST_HIT
GOALKEEPER_SAVE
```

Необходимо отказаться от кнопки `OUT` и унифицировать управление атакой.

---

# Цель задачи

Полностью убрать кнопку `OUT` с игрового экрана.

Теперь при наличии атакующей карты игрок всегда должен нажимать на одну из карт актуальной линии соперника:

```text
midfielders
defenders
goalkeeper
```

После клика движок должен определить результат:

1. Если атакующая карта может побить выбранную карту, сохранить существующую успешную атаку.
2. Если атакующая карта не может побить выбранную карту, завершить атаку как потерю мяча.
3. Если выбрана карта голкипера, сохранить существующую специальную логику удара по воротам.

---

# Новое поведение для полевых линий

Актуальными полевыми линиями являются:

```text
midfielders
defenders
```

Если игрок нажимает карту текущей линии, которую атакующая карта может побить:

```text
attackerCard beats defenderCard
```

нужно сохранить текущее поведение:

1. Запустить существующую анимацию движения атакующей карты:

```text
from active deck -> to target card
```

2. Показать существующий impact pulse.
3. Удалить побитую карту с поля соперника.
4. Добавить карты в `attackBank`.
5. Продолжить атаку.
6. Перерисовать актуальное состояние UI.

Если игрок нажимает карту текущей линии, которую атакующая карта не может побить:

```text
attackerCard cannot beat defenderCard
```

нужно:

1. Не показывать всплывающее сообщение об ошибке выбора.
2. Запустить анимацию движения атакующей карты к выбранной карте соперника.
3. Показать impact pulse или аналогичный короткий визуальный эффект столкновения.
4. Не удалять карту защитника с поля.
5. Не воспроизводить звук сэйва голкипера.
6. Завершить текущую атаку.
7. Отправить текущую атакующую карту в `attackBank`.
8. Вернуть карты из `attackBank` вниз колоды активного игрока согласно существующей логике завершения атаки.
9. Передать ход сопернику.
10. Показать существующее летящее сообщение:

```text
Мяч потерян...
```

11. После завершения анимации перейти к следующему ходу.

Важно: первое нажатие должно сразу фиксировать выбор игрока. После клика нельзя разрешать повторный выбор другой карты текущей линии.

---

# Поведение при выходе на вратаря

Для позиции:

```text
goalkeeper
```

необходимо сохранить текущую игровую логику без изменений:

```text
SHOT_ON_GOAL
GOAL_SCORED
GOALPOST_HIT
GOALKEEPER_SAVE
```

Также сохранить:

```text
sound-goal
sound-goalkeeper-save
sound-goalpost
```

Существующие правила гола, штанги и сэйва не менять.

---

# Изменение интерактивности карт

Сейчас кликабельными являются только карты, которые считаются легальными целями.

После изменения кликабельными должны стать **все занятые позиции актуальной линии соперника**, независимо от того, может атакующая карта побить выбранную карту или нет.

Пример:

```text
У соперника в полузащите лежат карты 5, 9 и K.
Атакующая карта игрока — 8.
```

Игрок должен иметь возможность нажать:

```text
5
9
K
```

Результаты:

```text
клик по 5 -> успешное пробитие
клик по 9 -> потеря мяча
клик по K -> потеря мяча
```

Не разрешать клики:

```text
по картам другой линии
по своим картам
по пустым слотам
во время анимаций
если атакующая карта еще не взята
```

---

# Важное UX-ограничение

Не добавлять рамку, контур, цветную подсветку или отдельное выделение всей активной линии защиты.

Не добавлять визуальные подсказки, которые заранее показывают игроку:

```text
какие карты можно побить
какие карты нельзя побить
```

Игрок должен самостоятельно оценивать карты и специальные правила.

Допустимо сохранить стандартное поведение отдельных интерактивных карт при наведении, если оно уже существует в проекте.

Не добавлять новый постоянный текст поверх футбольного поля.


Если текущая структура события не содержит нужных данных, аккуратно расширить ее.

Желательно передавать:

```ts
{

  positionId,

  defenderCard
}
```

Не менять существующую логику событий удара по воротам.

# Анимация неудачного единоборства

Использовать существующую систему анимации атаки в:

```text
src/scenes/GameScene.ts
```

Ориентироваться на текущую структуру:

```ts
createAttackAnimationContext(positionId)
getAttackAnimationOutcome(state, positionId)
showFlyingMessage(message, tone, onComplete?)
```

Для неудачного полевого единоборства можно добавить новый исход:

```ts
type AttackAnimationOutcome =
  | 'defeat'
  | 'miss'
  | 'goal'
  | 'post'
  | 'save';
```

Для:

```text
miss
```

нужно:

```text
запустить движение атакующей карты к цели
показать impact pulse
не удалять карту соперника визуально до перерисовки
не запускать звук
после анимации показать "Мяч потерян..."
завершить ход
```

Можно переиспользовать визуальную логику `save`, но без звукового эффекта голкипера и без текста:

```text
Goalkeeper!!
```

# Возможная архитектура изменений

Нужно адаптировать существующую реализацию, не переписывая движок полностью.

Ожидаемая логика метода выбора цели может выглядеть примерно так:

```ts
selectTarget(positionId: PositionId): void {
  const targetCard = this.getTargetCard(positionId);

  if (!targetCard) {
    return;
  }

  if (!this.isPositionInCurrentTargetLine(positionId)) {
    return;
  }

  if (positionId === 'goalkeeper') {
    this.resolveGoalkeeperAttack(positionId);
    return;
  }

  if (canBeat(this.state.currentAttackCard, targetCard)) {
    this.resolveSuccessfulAttack(positionId);
    return;
  }

  this.resolveFailedFieldDuel(positionId);
}
```

Это ориентировочный пример. Использовать актуальную структуру проекта и существующие методы.

Не дублировать код завершения атаки, если его можно переиспользовать.

# Тесты

Добавить или обновить тесты Vitest.

Обязательно покрыть сценарии:

## 1. Успешное пробитие карты полузащиты

```text
Атакующая карта может побить выбранную карту полузащиты.
```

Ожидаемый результат:

```text
карта соперника удалена
атака продолжается
карты добавлены в attackBank
ход не передан сопернику
```

## 2. Неудачная попытка пробить карту полузащиты

```text
Атакующая карта не может побить выбранную карту полузащиты.
```

Ожидаемый результат:

```text
карта соперника остается на поле
создается ATTACK_MISSED
атака завершается
attackBank возвращается вниз колоды активного игрока
ход переходит сопернику
```

## 3. Неудачная попытка пробить карту защиты

```text
Атакующая карта не может побить выбранную карту защитника.
```

Ожидаемый результат аналогичен потере мяча в полузащите.

## 4. Линия содержит и слабую, и сильную карту

Пример:

```text
Атакующая карта = 8
Карты актуальной линии = 5, 9, K
```

Проверить:

```text
выбор 5 -> успешное пробитие
выбор 9 -> потеря мяча
выбор K -> потеря мяча
```

## 5. После неудачного клика нельзя выбрать вторую карту

После первого выбора более сильной карты:

```text
ход должен завершиться
повторный selectTarget не должен изменить завершенный ход
```

## 6. Вратарь

Проверить, что после изменений сохраняются сценарии:

```text
гол
штанга
сэйв голкипера
```

## 7. Специальные правила карт

Убедиться, что продолжают работать:

```text
2 beats JOKER
6 beats A
7 beats K
8 beats Q
9 beats J
```

## 8. Кнопка OUT отсутствует

Если UI-тестов нет, проверить кодом, что:

```text
GameScene больше не создает кнопку OUT
```

# Критерии приемки

Задача считается выполненной, если:

1. Кнопка `OUT` полностью исчезла с игрового экрана.
2. После взятия атакующей карты можно нажать любую занятую карту актуальной линии соперника.
3. Нажатие карты, которую можно побить, сохраняет текущую успешную механику атаки.
4. Нажатие карты, которую нельзя побить, запускает визуальный эффект столкновения и завершает ход как потерю мяча.
5. При потере мяча карта соперника остается на поле.
6. При потере мяча не воспроизводится звук сэйва голкипера.
7. Показывается сообщение:

```text
Мяч потерян...
```

8. После первого клика нельзя пробовать нажать другую карту линии.
9. Логика вратаря, гола, штанги и сэйва остается рабочей.
10. Не добавлена рамка или отдельная подсветка активной линии защиты.
11. Не добавлены подсказки, заранее показывающие выигрышные карты.
12. Все существующие и новые тесты проходят.
13. Проект успешно собирается.
