# Total Soccer: Mundial - спецификация игрового экрана матча

Этот документ описывает только игровой экран матча: состав UI, расположение элементов, интерактивность, состояния, анимации и связь с кодом.

## 1. Назначение экрана

Игровой экран матча - это основная сцена партии, где игроки:

- видят футбольное поле и карты обеих команд;
- берут атакующие карты из своей колоды;
- выбирают карты соперника для пробития;
- завершают атаку через `OUT`;
- следят за счетом, ударами, голами и текущим превосходством;
- переходят в меню или на экран результата.

Основной файл:

```text
src/scenes/GameScene.ts
```

Основная сцена Phaser:

```text
GameScene
```

## 2. Размер сцены и базовые координаты

Глобальный размер сцены:

```text
SCENE_WIDTH = 1600
SCENE_HEIGHT = 720
```

Основные константы в `GameScene.ts`:

```text
FIELD_WIDTH = 1120

DECK_Y = 560

```

Центр сцены:

```text
centerX = 800

```

Игровое поле расположено по центру, его центр:

```text
x = 800
y = 400
```

## 3. Общая композиция экрана

Экран состоит из следующих зон:

1. Верхняя панель:
   - кнопка `В меню`;
   - табло счета;
   - кнопка `Результат`;
   - шкала текущего превосходства под табло.

2. Центральная зона:
   - футбольное поле;
   - карты первой команды слева;
   - карты второй команды справа;
   - пустые слоты, если позиция свободна.

3. Боковые зоны:
   - статистика первой команды слева;
   - статистика второй команды справа.

4. Нижняя зона:
   - колода активного/неактивного игрока слева;
   - колода активного/неактивного игрока справа;
   - кнопка `OUT` около колоды активного игрока.

5. Временные оверлеи:
   - всплывающие ошибки;
   - летящие сообщения `GOAL!!`, `Штанга!`, `Goalkeeper!!`, `Мяч потерян...`;
   - анимации движения карт.

## 4. Рендеринг сцены

Каждый рендер пересоздает динамический слой:

```text
dynamicLayer
```

Перед новым рендером старый слой уничтожается:

```ts
this.dynamicLayer?.destroy();
this.dynamicLayer = this.add.container(0, 0);
```

Это упрощает синхронизацию UI с `GameState`: сцена не обновляет отдельные элементы точечно, а перерисовывает актуальное состояние.

## 8. Отображение карт на поле

Карты отрисовываются через:

```text
src/ui/CardView.ts
```

Если позиция занята:

- отображается карта;
- у вратаря дополнительно показывается label `GK`;
- если карта является легальной целью, она становится кликабельной.

Если позиция пустая:

- отображается полупрозрачный слот `96 x 132`.

## 9. Легальные цели

Кликабельной может быть только карта соперника:

```ts
player.id !== state.activePlayerId
```

И только если ее позиция входит в:

```ts
state.legalTargetPositionIds
```

Легальные цели определяет игровой движок на основе текущей линии соперника.

## 10. Колоды игроков

Компонент:

```text
src/ui/DeckView.ts
```

Расположение:

```text
первая команда: x = 115,  y = DECK_Y
вторая команда: x = 1485, y = DECK_Y
```

Размер визуальной карты колоды:

```text
96 x 132
```

Колода показывает:

- рубашку колоды;
- количество карт рядом с колодой;
- текущую карту атаки, если она уже взята;
- анимированный маркер мяча над активной колодой.

Колода кликабельна только если:

```text
interactive = true
player is active
state.phase = WAITING_FOR_ATTACK_CARD
```

При клике:

```ts
drawAttackCard()
```

## 11. Кнопка OUT

Кнопка создается возле активной колоды:

```text
x = getDeckX(state)
y = OUT_BUTTON_Y
```

`getDeckX`:

```text
PLAYER_1 -> 115
PLAYER_2 -> 1485
```

Кнопка отключена, если:

```text
interactive = false
state.phase !== WAITING_FOR_TARGET
текущая цель - только goalkeeper
```

Во время удара по вратарю `OUT` запрещен.

При нажатии:

```ts
declareOut()
```

Результат:

- атака завершается;
- карта атаки уходит в `attackBank`;
- карты возвращаются вниз колоды активного игрока;
- ход переходит сопернику;
- показывается сообщение `Мяч потерян...`.


## 15. Анимация восстановления поля

После начала хода активный игрок восстанавливает пустые позиции поля картами из своей колоды.

События восстановления:

```text
FIELD_CARD_RESTORED
```

`GameScene` берет новые события восстановления из лога:

```ts
getRestoreAnimationEntries(state.log).slice(this.animatedRestoreCount)
```

Пока идет анимация:

```text
interactive = false
```

Карты летят:

```text
from deck position -> to restored field position
```

Параметры:

```text
duration = 420 ms
ease = Cubic.easeInOut
```

После каждой анимации увеличивается:

```ts
animatedRestoreCount
```

## 16. Анимация атаки

Когда игрок выбирает цель, сцена до изменения UI создает контекст:

```ts
createAttackAnimationContext(positionId)
```

Контекст содержит:

```text
attackerId
defenderId
positionId
attackerCard
```

После успешного выбора цели карта атаки летит:

```text
from active deck -> to target card
```

Параметры:

```text
duration = 340 ms
ease = Cubic.easeIn
```

После удара показывается impact pulse.

## 17. Исходы анимации атаки

Тип:

```ts
AttackAnimationOutcome = 'defeat' | 'goal' | 'post' | 'save'
```

Определение:

```ts
getAttackAnimationOutcome(state, positionId)
```

Если позиция не `goalkeeper`, исход:

```text
defeat
```

Для вратаря сцена смотрит последние события:

- `GOALPOST_HIT` -> `post`
- `GOALKEEPER_SAVE` -> `save`
- `GOAL_SCORED` -> `goal`
- иначе `defeat`

## 18. Звуки

Используются звуковые ключи:

```text
sound-whistle-start
sound-goal
sound-goalkeeper-save
sound-goalpost
```

Стартовый свисток играет один раз, когда:

```text
state.turnNumber === 1
startWhistlePlayed === false
```

## 19. Управление фазами через UI

Экран матча не принимает игровых решений самостоятельно. Он вызывает методы `GameEngine`:

```text
startNextTurn()
drawAttackCard()
selectTarget(positionId)
declareOut()
```

И затем реагирует на новое состояние:

- если `GAME_OVER`, открывает `ResultScene`;
- если `ENDING_TURN`, начинает следующий ход;
- если `WAITING_FOR_ATTACK_CARD`, ждет клика по активной колоде;
- если `WAITING_FOR_TARGET`, ждет выбора легальной цели или `OUT`.

## 20. Открытие экрана результата

Метод:

```ts
openResult(state)
```

Переход:

```ts
this.scene.start('ResultScene', { state })
```

Экран результата может быть открыт:

- автоматически при `GAME_OVER`;
- вручную кнопкой `Результат`.

## 21. Текущая шкала превосходства на экране матча

Шкала использует последние 5 ходов.

Очки глубины:

```text
полузащита = 1
защита = 2
вратарь = 3
```

За ход учитывается только максимальная глубина.

Максимальное давление за окно:

```text
5 ходов * 3 = 15
```

Формула:

```text
balance = clamp((playerOnePoints - playerTwoPoints) / 15, -1, 1)
playerOneShare = 0.5 + balance * 0.5
```

Визуально:

```text
0.5 = ровно 50/50
> 0.5 = преимущество первой команды
< 0.5 = преимущество второй команды
```

## 22. Важные UX-договоренности для будущих изменений

- Не добавлять лишний текст поверх игрового поля.
- Не дублировать названия команд в шкале превосходства.
- Шкала превосходства должна оставаться компактной.
- Карты и доступные цели должны оставаться главным визуальным фокусом.
- Ошибки выбора цели должны быть временными и не блокировать экран.
- Во время анимаций интерактивность должна отключаться.
- Кнопка `OUT` должна быть доступна только когда это разрешено правилами.
- Вратарь должен явно иметь label `GK`.
- Табло должно оставаться главным источником счета и названий команд.
